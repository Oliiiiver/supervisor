# Supervisor · 考公陪跑台

给备考朋友做的监督小工具。四个模块:

- **项目管理** — 考试倒计时、阶段路线(理论学习 → 专项刷题 → 查缺补漏 → 定时刷题)、双列今日任务(备考任务 | 陪跑任务)、逾期未完成账、今日全清横幅
- **奖励机制** — 完成任务赚积分(每日上限 100 分)、自定奖品目录兑换、积分流水
- **学习轨迹** — 累计数字(刷题量/学习天数/历史总积分,只涨不跌)、双人打卡热力图与共同打卡天数、成就墙(11 枚自动解锁的徽章)、专项刷题正确率走势、刷题/模考记录表
- **留言板** — 学习者对工具的需求和改进想法直接写在这里

纯静态站点,无构建步骤。数据层双模式:

| 模式 | 触发条件 | 数据存哪 |
|---|---|---|
| 本地模式 | `js/config.js` 留空(默认) | 本机 localStorage |
| 云端模式 | 填好 Supabase 配置 | Supabase,两人跨设备实时同步 |

## 分工

- **学习者(她)**:安排备考任务并打卡、录专项刷题记录、添加/兑换奖品、留言提需求
- **监督员(你)**:打开右上角「监督员模式」→ 给任务赋分值、给完成的任务和刷题记录点赞、销逾期账、推进阶段(点击阶段名)、设置考试信息、删任务、处理留言;同时在陪跑任务列给自己安排任务并打卡

> 监督员模式只是界面开关,不是权限控制——这是信任模型为"两个朋友"的小工具。

## 规则说明

- 备考任务分值由监督员赋(新任务默认"待赋分" = 0 分);陪跑任务不进积分经济
- 每日通过任务最多获得 **100 分**,超出部分照常打卡但不记分
- 取消勾选任务会撤回对应积分
- 逾期未完成的任务进入红色"之前未完成"区:可以补做(积分按补做当天的上限记),只有监督员能"销账"
- 成就徽章全部由数据自动判定;点赞只有监督员能点,学习者看到实心的赞就知道监督员来看过——这是有意为之的低调存在感

## 本地预览

直接双击 `index.html` 即可(或任意静态服务器)。默认本地模式,页脚有清空按钮。

## 接入 Supabase(云端同步)

1. 在 [supabase.com](https://supabase.com) 注册,新建一个免费项目
2. 打开项目的 **SQL Editor**,把 `supabase/schema.sql` 全部内容粘贴进去运行
3. 在 **Settings → API** 找到 `Project URL` 和 `anon public` key
4. 填进 `js/config.js`:

```js
window.CONFIG = {
  SUPABASE_URL: "https://xxxx.supabase.co",
  SUPABASE_ANON_KEY: "eyJhbGci...",
};
```

5. 刷新页面,页脚显示「云端模式」即成功

> 访问控制走的是"链接即权限"的简化方案(匿名可读写),网址别外传。
> 需要更严格时再给两人建 Supabase 账号、收紧 RLS 策略。

## 部署

推荐 [Vercel](https://vercel.com) 或 [Netlify](https://netlify.com):新建项目 → 拖入本目录(或关联 git 仓库)→ 完成。没有构建命令,输出目录就是根目录。

## 文件结构

```
index.html          页面骨架(四个 tab)
css/style.css       几何简洁风样式,自动适配深色模式
js/config.js        Supabase 配置(留空 = 本地模式)
js/demo-data.js     初始数据(四阶段、一个奖品,其余留空)
js/store.js         数据层(local/cloud 双模式,同一接口)
js/charts.js        专项刷题折线图(Chart.js)+ 打卡热力图(CSS grid)
js/badges.js        成就徽章定义与判定(图标为手绘几何 SVG)
js/app.js           页面逻辑
supabase/schema.sql 数据库结构,SQL Editor 里一键运行
```

## 设计说明

- 图表配色用的是经过色觉障碍(CVD)与对比度校验的分类色板;浅色模式下"言语/数量"两色对比度略低,已按规范用线端直接标注 + 数据表兜底
- 专项刷题走势图 y 轴统一为 0–100% 正确率,同一天同模块多组练习自动合并计算
- 模考记录保留框架,套题模考阶段再启用
